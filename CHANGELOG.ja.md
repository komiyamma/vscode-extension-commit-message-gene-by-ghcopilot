# Change Log

## 0.3.39

- 生成したコミットメッセージに `Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>` のような共著者トレーラーが入らないようにしました。

## 0.3.37

- 残っていたデバッグ出力をコメントアウトし、通常利用時に Output パネルへ大量の出力が出ないようにしました。

## 0.3.36

- 拡張の説明文を更新し、Codex 環境ではなく GitHub Copilot 環境を指すようにしました。

## 0.3.34

- ロゴ PSD を配布対象から除外し、公開される拡張に未使用の素材が含まれないようにしました。

## 0.3.33

- 提案 API の宣言を削除し、拡張のインストールや起動時に `contribSourceControlInputBoxMenu` を要求しないようにしました。

## 0.3.32

- Copilot CLI のパッケージ名判定を簡略化し、Windows では常に `@github/copilot-win32-x64` を使うようにして、拡張の対応環境と合わせました。

## 0.3.31

- `commitMessageGeneByGhcopilot.*` というこの拡張専用の名前空間に設定キーを分離し、Codex 版とはプロンプト設定を共有しないようにしました。

## 0.3.29

- 起動時に Copilot クライアントを事前ウォームアップし、workspace ごとに再利用するようにして、最初のコミットメッセージ生成が速く感じられるようにしました。
- Copilot の認証確認後に出していた一時的なログイン通知をやめ、出力やステータスバーが余計に出ないようにしました。

## 0.3.27

- 起動時に Copilot クライアントを事前ウォームアップし、workspace ごとに再利用するようにして、最初のコミットメッセージ生成が速く感じられるようにしました。

## 0.3.26

- README の要件を明確化し、この拡張を使う前に VS Code の GitHub Copilot か GitHub Copilot CLI のどちらかをインストール済み・ログイン済みで用意しておく必要があることを伝えるようにしました。

## 0.3.25

- VS Code エンジンと `@types/vscode` の対応範囲を `^1.100.0` に下げ、古い VS Code 互換エディタでも動作しやすいように調整しました。

## 0.3.24

- README を更新し、Cursor、Kiro、VSCodium、Antigravity などの VS Code 互換エディタ向けであることを明確にしました。

## 0.3.23

- コミットメッセージ生成を `@github/copilot-sdk` 0.2.2 に切り替えました。
- コミットメッセージ生成で使用するモデルを `gpt-5-mini` に更新しました。
